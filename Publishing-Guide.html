Publishing Guide
如何自己发布养生文章

Everything you need to add articles to the site yourself, after it goes live — set up once, then publish from a simple dashboard. AI can help you draft; this explains how it all fits together.

How it works · 运作方式

One file, one dashboard 一处编辑，全站更新

Every article lives in a single content file, content/articles.json. The website reads it, and the admin dashboard writes to it. You never touch code — you fill in a form and press Publish.

One-time setup · 首次设置（约 30 分钟）

Putting the site & dashboard online

Do this once. It's free, and best done by whoever is comfortable with websites (or hand this to a developer — it's a standard setup). After this, adding articles needs no technical help.

  1. Create a free GitHub account and upload this project's folder to a new repository (a "repo").
  2. Create a free Netlify account and choose “Add new site → Import an existing project”, then pick your GitHub repo. Netlify publishes your site at a free your-name.netlify.app address (you can connect a custom domain like xiastcm.com later).
  3. In Netlify, open Site configuration → Identity and click Enable Identity.
  4. Under Identity → Services → Git Gateway, click Enable Git Gateway. (This lets the dashboard save your articles.)
  5. Under Identity → Registration, set it to Invite only, then Invite your own email. Accept the email invite and set a password.
  6. Done. Visit your-site-address/admin/ and log in with that email — you'll see the article dashboard.

If any of the above feels unfamiliar, the “Handoff to Claude Code” package or any web developer can wire it up — the configuration files (admin/config.yml) are already in the project, so it's plug-and-play.

Adding an article · 发布一篇文章

The everyday part 日常操作

Once set up, this is all you ever do. Log into /admin/, click “养生资讯 · Articles” → New / 新增, and fill in these fields:

Field 字段What to enter 填写说明
SlugA short English web name, lowercase with hyphens — e.g. autumn-cough. Becomes the article's link.
分类 Categorye.g. 节气养生 / Seasonal Wellness. Both languages.
标题 TitleThe headline, in 中文 and English.
日期 DateDisplay text only — e.g. 2026 · 立秋 / 2026 · Early Autumn.
作者 Author夏 扬 / Xia, Yang — or 沈 宏 / Shen, Hong.
阅读时长 Reade.g. 约 4 分钟 / 4 min read.
摘要 ExcerptOne sentence shown on the article card.
正文 BodyThe full article. See formatting below.

Formatting the body 正文格式

The body is plain text with two simple marks:

  • Start a line with ## to make a subheading.
  • Start a line with > to make a highlighted note box (great for the “this is general information, please call to book” disclaimer).
  • Leave everything else as normal paragraphs. One line = one paragraph.
## 起居：晚睡早起 夏日昼长夜短，可顺应天时稍晚入睡、清晨早起…… ## 饮食：清淡为主 暑天易贪凉饮冷，反伤脾胃…… > 以上为一般养生参考，非诊断或治疗建议。如有不适，欢迎来电预约。

Article image 配图

After publishing, open the live article and drag a photo onto the image area at the top — it saves automatically. (Or upload via the dashboard's media button.)

Drafting with AI · 用 AI 起草

Let AI write the first draft

Open any AI assistant and ask for a bilingual draft you can paste straight into the fields. A prompt like this works well:

请以中医养生的角度，写一篇面向大众的科普短文，主题：「秋季润肺」。 要求：温和、通俗、不作疗效承诺；约 4 段，含 2–3 个小标题（用 ## 开头）； 结尾加一句温馨提示（用 > 开头），说明这是一般资讯、非诊断建议、欢迎来电预约。 最后附上同样内容的英文版本。
  • Ask for both 中文 and English in one go, so the two fields stay in sync.
  • Ask it to use ## for subheads and a > closing note — then it pastes in cleanly.
  • Keep it general and educational — no promises of cures or specific results.
Before you publish · 发布前

Always review first

!
AI drafts a first version — Dr. Xia has the final word 医师审核为准

This is health content from an Ontario-registered practitioner. Every article should be read and approved by Dr. Xia before publishing — both for clinical accuracy and because regulated health professionals have rules about claims and advertising. Avoid guarantees of results, naming specific outcomes, or anything that reads as a diagnosis. When in doubt, keep it general and end with “please call to book a consultation.”
